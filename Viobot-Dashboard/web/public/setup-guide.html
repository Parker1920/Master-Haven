Set up Viobot
on your server.

Viobot points at roles and channels that you create. This guide walks through exactly which roles, channels, and permissions to make so that moderation, mutes, tickets, and logging all work — then how to connect them with /setup, /config, or the dashboard.

Read this first

How Viobot uses your server

Understanding one thing up front saves the most common setup mistake.

Viobot does not change channel permissions for you. When it mutes someone, it simply gives them the Mute role — nothing more. That role only silences a person if you have set the channel permissions to deny it. So the most important step below is wiring the permissions — without it, muting adds a role that does nothing.

🎭 Roles

Viobot assigns roles (Mute, Voice Mute, Verification) and checks who is a Moderator. It never edits a role's own power — you decide that.

#️⃣ Channels

Viobot posts logs and violation records into channels you choose, and opens ticket threads inside a contact channel.

🔗 Config

/config (or the dashboard) just stores the IDs of what you made. Nothing is created automatically.

Prerequisites

Before you start

Two things to get right, or nothing downstream works.

Invite Viobot with moderation permissions

Viobot needs, at minimum: Manage Roles Manage Channels Manage Threads Kick Ban Moderate Members View Channels Send Messages Embed Links Read Message History.

If Viobot is already in your server, make sure these aren't removed by a restrictive role.

Put Viobot's role near the top of the role list

A bot can only manage roles that sit below its own highest role. Drag Viobot above every role it will assign — especially Muted, Voice Muted, and Verification Required.

This is the #1 reason mutes silently fail. If Muted is above Viobot in Server Settings → Roles, Viobot can't apply it.

Step 1

Create the roles

In Server Settings → Roles → Create Role. Give the moderation roles no special permissions — they're controlled by channel permissions in Step 3, not by their own power.

RolePurposeGive it permissions?
Moderator Whoever holds this role may run Viobot's staff commands (!ban, !mute, tickets, etc.). You can reuse an existing staff role instead of making a new one. Only the normal staff access you want them to have. It does not need Administrator.
Muted Viobot adds this to text-muted members. None. It silences people via the denies you set in Step 3.
Voice Muted Viobot adds this to voice-muted members. None. Silenced via a voice deny in Step 3.
Verification Required optional Marks members who still need to verify. Viobot respects it during mutes and when restoring roles to returning members. None here — how it gates access depends on your verification setup.

Only Moderator, Muted, and Voice Muted are required for a working setup. Add the others when you need them.

Step 2

Create the channels

Two categories keep things tidy: a private Staff area for logs, and a Contact area for tickets. Create them in Server Settings or by right-clicking the channel list.

🔒 Staff category hidden from @everyone

Staff-only channels for Viobot's output:

  • #mod-log — general moderation log
  • #violations — one entry per warn / mute / ban
  • #papertrail — record trail for audit

📮 Contact category

Where members reach staff and tickets open:

  • #contact-staff — members open tickets here (Viobot creates a private thread)
  • #muted-contact — the one channel a muted member can still use to reach staff

Each channel maps to one Viobot setting:

ChannelMaps to config fieldWho posts / uses it

#violationsviolationsChannelIdViobot posts each violation record
#papertrailpapertrailChannelIdViobot posts audit trail
#contact-staffcontactMemberChannelIdMembers open tickets; Viobot opens private threads
#muted-contactcontactMutedChannelIdMuted members contact staff
Step 3 — the important one

Wire the permissions

This is what actually makes mutes silence people and keeps staff channels private. Set overwrites on the category where possible — channels inside inherit them automatically.

How Discord inheritance works: a permission you set on a category applies to every channel inside it, unless a channel overrides it. So denying the Muted role on each category covers all the channels under it in one move. Channels that sit outside any category need the deny set directly.

🔇 Make the Muted role actually mute

On every category (and any channel not in a category), edit permissions → add the Muted role → set these to deny (❌):

PermissionMuted roleWhy
Send Messages❌ DenyCan't talk in text channels
Add Reactions❌ DenyCan't react as a workaround
Create Public Threads❌ DenyCan't spin up new threads
Create Private Threads❌ DenySame, private threads
Send Messages in Threads❌ DenyCan't talk inside existing threads
Speak❌ DenyCan't talk in voice channels

The one exception: on #muted-contact, override the above — set the Muted role to ✔ Allow View Channel and Send Messages. This leaves muted members exactly one place to reach staff.

🔊 Make the Voice-Muted role mute voice

On each voice category / voice channel, add the Voice Muted role and set:

PermissionVoice Muted roleWhy
Speak❌ DenyStays in voice but can't be heard

On the Staff category:

Step 4

Register & connect Viobot

Now tell Viobot about everything you just made.

Register the server

Run /setup once. This adds your server to Viobot's database so it can hold a configuration. It does not create anything.

Point Viobot at your roles & channels

Run /config and use the role and channel pickers — or open the Viobot Dashboard and set them there. Match each picker to what you built using the cheat sheet below.

Turn on the features you want

In /config or the dashboard, enable:

  • Tickets — the contact-us / private-thread ticket system.
  • Logger — moderation + server logging into your log channels.
  • Auto-susmute new accounts — automatically suspicious-mutes very new accounts on join.

Step 5

Test it before you rely on it

Run through this on a test account (or with a willing staff member) to confirm everything is wired.

  • Warn the test user — a record appears in #violations.
  • Temp-mute the test user — they get the Muted role and genuinely can't send messages anywhere.
  • The muted user can still post in #muted-contact.
  • Voice-mute the test user — they can join voice but not be heard.
  • Open a ticket from #contact-staff — a private thread appears with the member + staff in it.
  • Confirm regular members cannot see the Staff category.
  • Unmute — the role is removed and they can talk again.
Troubleshooting

Common problems

😶 Muting doesn't silence anyone

Either the Muted role is above Viobot in the role list (drag Viobot higher), or you haven't set the deny permissions from Step 3. The role by itself does nothing.

🚫 Viobot posts nothing in the log channels

The Staff category hides channels from @everyone — but you must also Allow the Viobot role View + Send Messages + Embed Links there.

🎫 Tickets fail to open

Viobot needs Create Private Threads and Manage Threads in #contact-staff, and the Tickets feature must be enabled in config.

🔕 A muted member can't reach staff

Check the #muted-contact exception: the Muted role must be Allowed View + Send there, overriding the category-wide deny.

Changed settings in the dashboard and Viobot hasn't picked them up? Viobot caches each server's config — a change may take a short moment (or a bot reload of that server) to apply. Roles and channels you created in Discord always take effect immediately.
