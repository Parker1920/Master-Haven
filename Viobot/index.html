What Viobot does

Viobot is built around practical server moderation: fast actions, clean records, fewer repeated tasks, and better visibility for staff teams.

MOD

Moderation actions

Warn, mute, voice mute, ban, unban, set nicknames, update roles, and keep a clear record of what happened and why.

LOG

Logs and accountability

Send moderation records and safety alerts to staff channels so decisions are easier to review later.

ROL

Role restoration

Save roles during mutes or when members leave, then restore what Viobot can safely manage when appropriate.

TKT

Tickets and contact flows

Give members and muted users a controlled way to contact staff without opening up the whole server. Plus/Beta servers can add custom Contact Us categories.

ALX

Public and private aliases

Create reusable message shortcuts, embeds, and moderation helper responses for your staff team.

SEC

Safety automation

Detect suspicious scam-style image posts, flag unusual activity, and help staff respond faster.

AM

Discord AutoMod visibility

Use Viobot alongside Discord AutoMod so staff can review important automated safety events in the same moderation workflow.

SCH

Timed announcements

Schedule one-time announcements to post later in selected channels, then list or cancel pending announcements before they send.

PLS

Plus and Beta command access

Keep premium and beta tools separate from regular moderation commands, with !plustools showing whether access is through Plus, Beta, both, or locked.

Quick start

A simple setup path for server owners and staff. Exact permissions may depend on your server layout.

Add Viobot and place its role correctly. Viobot needs its role above the roles it should manage, especially mute, verification, and moderation-related roles.
Register the server, then configure it. Run /setup once to add your server to Viobot's database. Then use /config to choose roles, channels, and feature settings.
Choose staff-only channels. Use /config to set where logs, violation records, papertrail links, and contact or ticket messages should go.
Test common actions. Try a warning, a temporary mute, a ticket flow, a scheduled announcement, and a role restore test before relying on Viobot for a large server.

Free, Plus, and Beta access

Viobot's standard moderation tools remain available without Plus. Viobot Plus and approved Beta access currently unlock moderation activity reports, higher alias limits, custom Contact Us ticket categories, and timed announcement commands.

Area Standard Viobot Viobot Plus / Beta Access
Core moderation Warns, mutes, bans, role changes, nickname changes, tickets, logging, reminders, and moderation records. The same core moderation tools. Plus/Beta does not replace the regular staff workflow.
Moderation reports Staff can review logs and individual moderation records. /activityreport summarizes recent moderation activity, staff activity, repeat users, and recent actions.
Alias limits Public and private aliases are available with standard limits. Higher public and private alias limits for larger servers and staff teams that rely on reusable responses.
Timed announcements Standard reminders remain available for personal reminders. !announce schedules server announcements, lists pending announcements, and cancels scheduled announcements before they send.
Custom ticket categories Contact Us tickets use the default categories, such as report and other. /contactcategories can add custom Contact Us categories for servers with access.
Access visibility !modtools lists regular staff tools. !plustools lists premium/beta commands and shows whether access is through Plus, Beta, both, or locked.
Beta access is used for approved testing. Plus and Beta access can unlock individual feature flags, and !plustools shows the current server's access state for premium/beta message commands. Features are listed here only after they exist in Viobot.

Discord AutoMod integration

Viobot is designed to work alongside Discord AutoMod, not replace it. AutoMod can catch or block content automatically, while Viobot helps staff keep those events visible and organized.

AM

Review automated safety events

When Discord AutoMod takes action, Viobot can send useful context to the configured logging channel so staff can review what happened without digging through server settings.

LOG

Keep logs in one staff workflow

AutoMod alerts can sit beside Viobot's moderation logs, scam alerts, role changes, tickets, and other staff-facing records.

1
Configure Discord AutoMod in Server Settings. Set your server's keyword, spam, mention, or safety rules using Discord's built-in AutoMod tools.
2
Use /config to choose Viobot's logging channel. Viobot uses the configured log channel to surface AutoMod events and other server safety activity for staff.
3
Let staff review and follow up. AutoMod handles the automatic rule match. Viobot helps staff see the event, discuss it, and take follow-up moderation action when needed.

Common workflows

These examples show how Viobot fits into normal staff work without needing to understand the internals.

01

Temporary mute

Staff runs !mute 2h @User Spamming after warning. Viobot saves manageable roles, applies mute roles, logs the action, and can restore roles after the mute expires.

02

Compromised account response

Staff runs !mutespammer @User or !bancompromised @User. Viobot uses the server's saved safety message and records the action for later review.

03

Appeal or staff note

Staff replies to a Viobot violation log to add context. Starting the reply with = replaces the reason instead of appending a new note.

04

Reusable staff response

Staff creates an alias once, then uses it later as a quick response for repeated situations like contact instructions, scam warnings, or ticket guidance.

05

Scheduled announcement

Staff runs !announce 2h #announcements Event starts soon. Viobot stores the announcement, posts it later, and lets staff list or cancel pending announcements.

06

Custom contact category

Staff uses /contactcategories to add a server-specific ticket option. Members then choose that category from the Contact Us menu when opening a ticket.

FAQ

Quick answers for server staff and members.

Does Viobot replace moderators?

No. Viobot helps staff act faster and keep better records, but moderation decisions are still made by server staff or by the server's configured automations.

Why did a mute not remove every role?

Discord does not allow bots to remove certain managed roles or roles above the bot. Viobot is designed to skip roles it cannot safely manage instead of breaking the whole action.

Who can use moderation commands?

Most moderation commands require the moderator role configured for that server. Server owners and staff control which role that is.

How do staff see Plus or Beta command access?

Moderators can use !plustools to see premium and beta message commands separately from !modtools. The list shows whether each command is locked or unlocked through Plus, Beta, or both.

Can Viobot schedule announcements?

Servers with access can use !announce to schedule one-time announcements, list pending announcements, and delete or cancel announcements before they send.

Does Viobot store data?

Viobot stores the information needed for its features, such as server configuration, moderation records, reminders, aliases, ticket metadata, role snapshots, and log references.

Can a server change what Viobot logs?

Yes. Server staff can use /config to choose logging channels and feature settings. /setup only registers the server so those settings can be saved.
