## Pass 5 — Implementation Details Not in Planning Docs (⚙️)

### File layout conventions (observed)

- **Backend**: 12 `APIRouter` modules under `Haven-UI/backend/routes/` after the v1.50.0 March refactor (from 18,752-line monolith). Services under `Haven-UI/backend/services/`. Shared constants in `constants.py`, DB helpers in `db.py`.
- **Backend data files**: `Haven-UI/backend/data/galaxies.json` — bundled because Pi deployment doesn't include `NMS-Save-Watcher/data/`. Galaxy lookups break without this file (v1.38.5 fix, 2026-02-28).
- **Frontend hooks**: `Haven-UI/src/hooks/useDebounce.js`, `useDateFormat.js` — extracted from 3+ duplicates each in v1.48.0 refactor.
- **Frontend navigation**: `NAV_LINKS` + `NAV_GROUPS` arrays drive both desktop and mobile views (v1.48.0). Adding a route means touching arrays only.
- **Extractor**: `NMS-Haven-Extractor/dist/HavenExtractor/mod/` contains runtime code. Vendored `nms_namegen/` (v1.9.0) sits alongside `haven_extractor.py`, `nms_language.py`, `structs.py`, `pymhf.toml`.
- **Extractor distribution**: two zip types. Full (~112 MB, includes embedded Python) for new users. Mod-only (~60 KB) for auto-updater.

### Naming conventions (observed)

- **Backend**: snake_case functions, `route_handler_name` pattern. Endpoints mirror function names with dashes. Sub-admin scoping uses `discord_tag` parameter threaded through. Self-submission checks use `check_self_submission()` (single helper replacing 5 duplicated blocks).
- **Frontend**: PascalCase components (`AdvancedFilters.jsx`, `CommunityDetail.jsx`). camelCase hooks (`useDebounce`). Utilities live at `Haven-UI/src/utils/`.
- **Extractor**: `_private_helper()` prefix for internal functions. Version tags (`v1.6.14`) in class docstrings, incremented per micro-change.

### Error handling patterns

- **Backend**: FastAPI `HTTPException(status_code, detail)` for API errors. Approval endpoints do targeted DELETEs before INSERTs to avoid duplicate-on-resubmit bugs (the moons-duplication fix in v1.48.1 is representative).
- **Extractor**: Graceful degrade pattern everywhere — if a primary read fails, fall through to struct-access, then to hardcoded defaults. `_is_unresolved()` treats both `"Unknown"` and `"Unknown(N)"` as failure to trigger fallbacks.
- **Extractor**: `_resolve_adjective(text_id, field_type)` centralizes PAK→Translate→raw-text-id fallback chain.

### Configuration

- **Extractor user config**: `~/Documents/Haven-Extractor/config.json` — holds per-user API key + Discord tag + reality + game mode.
- **Extractor communities cache**: `~/Documents/Haven-Extractor/communities_cache.json` — refreshed from `/api/communities` on startup, falls back to hardcoded defaults if API unreachable.
- **Extractor adjective cache**: `~/Documents/Haven-Extractor/adjective_cache.json` — built from NMS PAK files on first run, rebuilt when PAK mtime changes.
- **Haven-UI `.env`**: frontend API base URL only.
- **Backend secrets**: Super admin settings in `super_admin_settings` table (DB), not env vars. Session tokens in `sessions` table.

### Auth implementation

- **Session check**: `get_session(session)` returns session_data dict with `user_type`, `discord_tag`, `account_id`, `profile_id`, `enabled_features`. Profile-based unified auth via `user_profiles` table is primary as of v1.48.0; legacy `partner_accounts` / `sub_admin_accounts` kept as fallback.

## Pass 6 — Execution Lessons (📚)

Operational wisdom that only lives in Claude Code transcripts.

### Build / deploy

- **Docker Desktop entries in `hosts` file are leftover** from when Docker was running on the PC. Docker no longer installed locally; entries are dead. (MEMORY.md already captured.)
- **xFi Advanced Security silently blocks local device-to-device traffic** — root cause of March 2026 Pi connectivity outage. Check Xfinity app → Internet → Advanced Security → unblock the PC if LAN reach fails.
- **Xfinity gateway hairpin NAT not supported** — `havenmap.online` from inside the LAN requires a hosts entry `10.0.0.229 havenmap.online`.
- **Pi deployment needs `galaxies.json` bundled** at `Haven-UI/backend/data/` — the original location under `NMS-Save-Watcher/data/` isn't deployed. Broke every non-Euclid galaxy edit on Pi until v1.38.5.
- **Auto-updater for extractor** uses GitHub Releases mod-only zip. Naming convention: `HavenExtractor-mod-v{VERSION}.zip`. Old zips archived to `NMS-Haven-Extractor/archive/`.
- **hgpaktool auto-install** must use embedded Python path (`Path(__file__).parent.parent / "python" / "python.exe"`), NOT `sys.executable` — inside pyMHF `sys.executable` returns `NMS.exe`, triggering game restart. (v1.6.3 fix, 2026-02-28.)

### Database / migrations

- **Migration failures leave an ambiguous state** — the backup is taken but the partial migration's DDL is rolled back at SQLite level only if the failing statement raises. Recovery: restore from `pre_migration_*.db`.
- **Do NOT skip hooks** (`--no-verify`) when a pre-commit hook fails — create a new commit after fixing, not an amend. (Parker's durable preference, captured in CLAUDE.md.)
- **`is_complete` column repurposed from boolean → 0-100 score** in v1.34.0. Reading old code that treats it as boolean will silently break completeness display.
- **Galaxy 0-indexed vs 1-indexed trap**: community says "Galaxy 256" (1-indexed). Code often uses `galaxies.json` entries indexed at 0 (`"0": "Euclid"`, `"255": "Odyalutai"`). Fallback default `"Euclid"` masks off-by-one errors.

### Frontend / refactoring

- **March cleanup refactor (v1.50.0)** did land — removed ~500 lines of duplicated adjective mapping tables, extracted 12 route modules from monolith, removed duplicate `useDebounce`/`useDateFormat` inlines, replaced PlanetEditor+MoonEditor with unified CelestialBodyEditor. Audit prompt language suggests it "claimed done but hadn't landed" — transcripts show it did land in March.
- **`useEffect` firing on mount with empty string** can overwrite loaded values — GlyphPicker was silently clearing glyph codes on edit (v1.45.3 fix, 2026-03-17). Guard with `useRef` for "skip first empty emission."
- **WebP Content-Type on Pi**: Starlette's `StaticFiles` serves `.webp` as `text/plain` on systems where Python's `mimetypes` lacks the mapping. Call `mimetypes.add_type("image/webp", ".webp")` at startup. (v1.45.1 fix.)
- **Editing `control_room_api.py`** (was 18,752 lines before v1.50.0 split) requires reading exact whitespace before `Edit` — tab vs space mismatches cause "String not found."

### Extractor / NMS reverse engineering

- **NMS struct layouts shift between game updates.** Voyagers broke `cGcPlayerState.mLocation.GalacticAddress` — nested struct fields returned 0. Switched to `mPlanetDiscoveryData.mUniverseAddress` (single packed uint64, single offset, lower exposure to future breakage). Always prefer packed-uint64 reads over nested struct reads when both exist.
- **Adjective display strings only populate AFTER game's Translate function caches them.** Capture time ≠ extraction time. `_auto_refresh_for_export()` is the only reliable source; batch systems that weren't the currently-loaded system get stale data unless APPVIEW hook or user-triggered Apply Name forces a refresh.
- **PlanetInfo.Flora/Fauna/Sentinel/Weather are EMPTY at hook time.** Don't trust them in `on_creature_roles_generate`.
- **Hook fires in GenerateCreatureRoles order ≠ memory slot order** — don't use array index to correlate captured entries. Key by planet name.
- **`INHABITING_RACE` raw > 6 = "no data available for this system"** (legitimate game state).
