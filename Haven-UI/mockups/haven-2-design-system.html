Haven 2.0 — Unified.

Everything below assumes the Systems, Wizard, and Landing redesigns are the reference. This mockup shows the shared language extracted from those pages, plus how it'd land on the rest of the site. Open it next to havenmap.online and treat each block as a target.

How to read this: Tokens at top establish the palette. The card patterns are reproductions of what the Systems and Regions pages already do. The Hero / Stat tiles / Form / Table / Live-Preview blocks are new chrome applied to existing screen archetypes — they're what Profile, Settings, Analytics, Events, UserManagement should look like after a unification pass. The tier map at the end shows which existing pages each archetype unblocks.

07Coverage — what this unlocks

Each archetype above replaces the dated chrome on a cluster of existing pages. Sequencing: do the shared components first (Card / Button / FormField / StatTile / TableContainer), then apply the new page archetypes top-down by impact.

Tier A · Cosmetic

9 pages
  • Systems, Wizard, Landing (reference)
  • Changelog, Docs (already 2.0)
  • Dashboard — apply Hero + Stat tiles
  • CommunityStats — glass-ify community grid
  • CommunityDetail — glass region grid
  • Discoveries — hero + glass type cards
  • SystemDetail — hero block + glass planets
  • PendingApprovals — glass per-submission
  • RegionDetail — hero + glass system grid

Tier B · Restyle

8 pages
  • Analytics — glass tables, Cinzel headers
  • PartnerAnalytics — same as Analytics
  • Events — glass event cards + accent badges
  • Profile — form panel pattern
  • DBStats — glass table pattern
  • CsvImport — glass drop zone + form sections
  • DataRestrictions — glass toggle groups
  • PartnerManagement — glass modals

Tier C · Full Rebuild

9 pages
  • WarRoom — palette rethink, glass war cards
  • WarRoomAdmin — same
  • ApiKeys — glass CRUD + hero
  • ApprovalAudit — filter hero + glass results
  • Settings — form panels + accent active state
  • SubAdminManagement — glass user grid
  • UserManagement — glass user grid
  • ExtractorUsers — full glass + glow
  • Logs — dark glass log viewer
SEQUENCING NOTE — The fastest path to "site feels unified" is upgrading the shared components first: Card, Button, FormField, StatCard, and a new TableContainer. That alone moves every Tier A and most Tier B pages 60–70% of the way without touching any individual page. The Tier C work (WarRoom, ApiKeys, Settings, Audit, the user-management trio) is where actual page rebuilds happen — those are the 1–2 week investments.

IDENTITY — The brand is already coherent: cosmic-compass mark + Cinzel wordmark + teal/purple/amber. The 2.0 pass is execution, not rebrand. The one thing worth deciding now: do we want the wizard live-preview to ship (the v1.1 mockup screenshot) — because that's a forward-looking feature, not a restyle, and it's the only "new pattern" in the queue.
